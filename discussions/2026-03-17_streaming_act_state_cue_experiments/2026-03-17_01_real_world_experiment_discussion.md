# 2026-03-17 Streaming Path Signature Enhanced ACT 真实场景实验讨论整理

## 1. 讨论背景

本轮讨论的目标是为 `streaming path signature` 增强的 ACT 设计真实场景实验任务，重点验证其在长时序记忆上的优势。

当前实验平台为：

- 一套带轮式移动底盘的双臂机器人
- 每个手臂末端带腕上相机
- 机器人头顶有一个全局视角相机

初始目标是希望任务足够炫酷、新颖，并且能够明显体现相较于普通 ACT 的优势。

## 2. 当前方法约束

讨论过程中逐步明确了一个核心约束：

- 当前 `streaming path signature` 主要只能对**系统状态序列**做长序列记忆编码
- 对图像观测如何做长期记忆，当前还没有确定方案

因此，后续任务设计必须满足：

- cue 不能主要依赖图像
- cue 必须能直接体现在系统状态中
- 系统后续必须经过一段共享流程，使得当前状态和当前视觉尽量趋同
- 真正的 disambiguation 必须依赖历史状态记忆，而不是当前观测

## 3. 第一轮备选方案

第一轮提出过几类真实任务方向：

1. 双分叉线缆编织与插接
2. 移动式空间站维修
3. 对称货架拣选加双臂装配
4. 瞬时配方的移动调饮/调剂任务
5. 镜像房间搜救与返回

这些方案整体思路是正确的，即：

- 让任务包含早期 cue
- 中间加入长时间共享段
- 末端设置延迟分叉或延迟执行

但用户指出上述方案仍有不足：

- 许多方案中的 cue 仍然默认依赖视觉信息或环境中的显式提示
- 这与当前 `path signature` 只能稳定编码系统状态的现实不完全匹配

因此，设计方向随后被收紧为：

- 寻找**系统状态本身就能承载 cue** 的任务

## 4. 约束收紧后的设计原则

在这一轮收紧后，讨论得到的设计原则为：

### 4.1 cue 必须进入系统状态

可作为 cue 载体的状态包括：

- 移动底盘位姿与轨迹
- 双臂关节角
- 双臂末端位姿轨迹
- gripper 开合状态

### 4.2 任务结构应采用如下模板

建议任务满足：

`状态编码段 -> 长共享段 -> 同态终局分叉 -> 不可逆结果`

换句话说：

- 早期通过系统状态写入任务信息
- 中间通过共享路径或共享操作“洗掉”当前状态差异
- 末端通过一个或多个分叉决策检验模型是否真正记住早期历史

### 4.3 必须设计 canonical merge pose

如果不同任务的状态差异一直保持到终局，那么模型不需要长期记忆。

因此应明确设计：

- 同一个底盘标准位姿
- 同一个双臂标准中间姿态
- 尽量相同的当前视觉外观

这样终局决策才真正依赖历史。

## 5. 讨论中最感兴趣的方向

在第二轮讨论中，用户认为最有趣的方案是：

**双臂导环穿线编码 + 四端口插接**

该方案的优点在于：

- 任务过程具有明显的“路径感”
- 双臂操作有展示性
- 天然适合和 braided hub 类任务建立呼应
- 如果设计合理，可以让 cue 完全进入系统状态而不依赖图像

## 6. 该方案的第一版思路

第一版思路为：

- 机器人抓住一根线束/软管
- 经过两次导环分叉，例如上/下、上/下
- 两次分叉形成一个 2-bit route code
- 最后在 4 个对称端口中选择对应端口完成插接

这版方案的核心逻辑是：

- 两次 routing 的历史承担 cue
- 普通 ACT 在长共享段后容易忘记早期 route
- signature 增强 ACT 则更适合保留这段长时序信息

## 7. 用户提出的进一步要求

用户随后指出，仅仅让“早期做出不同 routing”还不够，希望进一步加强：

- 系统状态在**一开始**就能分成 4 类
- 这 4 类初始状态应能更直观地指向下一步运动方向
- 从 episode 起始阶段开始，signature 就应体现出明显差异

这意味着任务需要从“早期动作编码”进一步收敛到：

- **四态起始编码**

## 8. 当前收敛出的推荐版本

### 8.1 方案名称

**四态预加载穿线 -> 汇流等化 -> 双分叉导环 -> 四端口插接**

### 8.2 核心装置

建议在机器人前方布置两个“发线塔”：

- 左塔：有 `Upper` / `Lower` 两个起始卡槽
- 右塔：有 `Upper` / `Lower` 两个起始卡槽

reset 时：

- 左臂和右臂分别抓住同一根线束的两个固定抓点
- 左腕放在左塔的 `Upper` 或 `Lower`
- 右腕放在右塔的 `Upper` 或 `Lower`

这样就得到四种初始系统状态：

- `S00 = L-Up, R-Up`
- `S01 = L-Up, R-Low`
- `S10 = L-Low, R-Up`
- `S11 = L-Low, R-Low`

这些初始状态直接体现在：

- 双臂关节角
- 双腕末端空间位置
- 双臂相对构型
- 可能的轻微底盘对齐状态

### 8.3 与后续任务语义的绑定方式

当前推荐的绑定方式是：

- 左臂初始高/低，决定第一次导环 `H1` 走上环还是下环
- 右臂初始高/低，决定第二次导环 `H2` 走上环还是下环

因此：

- 左塔 `Up/Low` = 第 1 bit
- 右塔 `Up/Low` = 第 2 bit

这样早期状态不是任意标签，而是与未来路由规则一一对应。

### 8.4 为什么“下一步方向”会更直观

在这个版本里，起始状态不是抽象编码，而是与第一步动作方向直接相关：

- 左臂从高槽出发，第一段脱槽动作天然更接近“向上/向上方导向”
- 左臂从低槽出发，第一段脱槽动作天然更接近“向下/向下方导向”
- 右臂同理

因此：

- 模型在最开始就能从系统状态中感知“应该朝哪个方向起步”
- 起始 prefix 的 signature 也能更早形成区分

## 9. 关键模块：等化门

为了防止任务退化为“当前状态就能看出来”，当前方案中加入了一个非常关键的模块：

**等化门**

等化门的作用是：

- 在线束经过中央导向梳 / funnel / tensioner 后
- 把 4 类任务重新汇流到同一个标准中间姿态

理想效果是：

- 底盘到达统一标准位姿 `B*`
- 双臂到达统一标准姿态 `A*`
- 当前视觉外观尽量相同

这样在等化门之后：

- 当前状态不再提供任务 id
- 唯一可用的信息变成历史状态轨迹

这正是 streaming path signature 应该发挥作用的位置。

## 10. 当前版本的完整任务流程

当前推荐把任务拆成 5 段：

### 10.1 四态起始

- 机器人以 `S00/S01/S10/S11` 之一开始
- 双腕落在不同高低卡槽中

### 10.2 发线脱槽

- 双臂先把线束从初始卡槽中提出
- 这一段就已经形成 4 类不同的早期状态轨迹

### 10.3 等化汇流

- 线束通过中央等化门
- 机器人进入统一标准中间姿态

### 10.4 双分叉导环

- 在 `H1` 做第一次上/下选择
- 经过一段共享段
- 在 `H2` 做第二次上/下选择

### 10.5 四端口插接

- 最后到达 2x2 的四个对称端口 `P00/P01/P10/P11`
- 目标端口由 `(H1, H2)` 或 `(起始左高低, 起始右高低)` 唯一决定

## 11. 当前版本相较于第一版的提升

当前版本比第一版更强，主要体现在三点：

### 11.1 cue 在 t=0 就已进入系统状态

不需要等机器人先做一段 routing，任务信息从起点就已经写进：

- 双臂构型
- 双腕位姿
- 双臂相对几何关系

### 11.2 cue 与未来动作方向具有结构对应

不是“4 个 arbitrary label”，而是：

- 左起始高低 -> 第一次导环方向
- 右起始高低 -> 第二次导环方向

### 11.3 经等化门后才真正考验长期记忆

这使任务更适合用来检验：

- 普通 ACT 是否会忘掉早期状态
- signature 增强 ACT 是否能把这段历史持续保留下来

## 12. 工程实现上的建议

讨论中给出的工程建议包括：

- 线束不要太软，优先使用半柔性 cable / hose / braided rope
- 起始卡槽先做简单的高/低两层，不要一开始就做复杂空间拓扑
- `H1/H2` 先做大尺寸上下导环，降低接触不确定性
- 末端插接可以先做磁吸或锥形导向的假插接
- 等化门结构应尽量刚性、简单、可重复

## 13. 建议重点评估的指标

当前讨论中认为，最重要的不是单一总成功率，而是分阶段记忆指标：

- `H1` 选对率
- `H2` 选对率
- 最终端口 `P00-P11` 选对率
- 错误发生位置：第一次分叉 / 第二次分叉 / 最终插接
- 等化门后共享段加长时，性能如何变化

其中一个重要建议是做：

**memory-gap sweep**

例如把等化门后的共享段长度设置成：

- `2s`
- `5s`
- `10s`
- `15s`

观察：

- 普通 ACT 的性能如何衰减
- signature 增强 ACT 的优势如何随记忆跨度扩大而被放大

## 14. 当前最适合继续推进的方向

截至本轮讨论，最值得继续推进的主方案是：

**四态预加载穿线 -> 汇流等化 -> 双分叉导环 -> 四端口插接**

其原因是：

- cue 已被彻底转移到系统状态
- 起始 signature 可尽早体现差异
- 中间存在明确的状态汇流与记忆洗平
- 任务展示效果强，且与当前方法高度同构

## 15. 建议的后续工作

后续最自然的推进顺序为：

1. 明确系统状态向量的最终组成
2. 画出场地布局与装置草图
3. 定义四种 reset 状态的精确几何参数
4. 设计等化门结构
5. 设计 `H1/H2` 与 `P00-P11` 的物理布局
6. 规划示教/采集 demo 的方式
7. 制定 baseline 与 ablation
